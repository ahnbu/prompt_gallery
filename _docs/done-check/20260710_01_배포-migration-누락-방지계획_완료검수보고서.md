- Impact: 만약 Wrangler의 출력이 '적용된 migration'과 '미적용 migration'을 함께 나열하는 형태라면, 이미 적용된 파일 이름도 `stdout.includes(name)`에 의해 true로 평가되어 Pending 목록에 들어갑니다. 누락이 발생했을 때 배포를 차단(FAIL)하는 핵심 목표는 달성되지만, 에러 메시지(`Pending remote D1 migrations: ...`)에 잘못된 파일이 포함되어 디버깅 시 혼선을 줄 수 있습니다.
- Severity: Advisory
- Fix: 추후 CLI 명령어에 `--unapplied` 플래그를 추가(`wrangler d1 migrations list <db> --remote --unapplied`)하여 미적용 목록만 안전하게 받아오거나, 정규식을 사용하여 출력 문자열 파싱을 더 엄격하게 개선할 것을 권장합니다.
