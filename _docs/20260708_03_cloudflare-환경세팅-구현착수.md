## 구현 설정값

`wrangler.jsonc` 또는 `wrangler.toml`에는 아래 의미로 바인딩한다.

```json
{
  "d1_databases": [
    {
      "binding": "DB",
      "database_name": "prompt-gallery-db",
      "database_id": "138200be-9d3b-4acf-bb71-42d5ca7e43b7"
    }
  ],
  "r2_buckets": [
    {
      "binding": "PREVIEWS",
      "bucket_name": "prompt-gallery-previews"
    }
  ]
}
```

Worker 코드에서는 다음 이름을 사용한다.

- D1: `env.DB`
- R2: `env.PREVIEWS`
- 이미지 preview 접근: public bucket이 아니라 Worker proxy 경로 `/api/assets/:id/content`

## 구현 시작 명령 초안

실제 구현 시작 시에는 먼저 스캐폴드와 설정 파일을 만든다.

```powershell
cd D:/vibe-coding/prompt-gallery

pnpm add react react-dom lucide-react
pnpm add -D typescript vite @vitejs/plugin-react @cloudflare/vite-plugin wrangler @types/react @types/react-dom
```

초기 구현 완료 후 검증 명령은 다음을 기준으로 둔다.

```powershell
pnpm typecheck
pnpm lint
pnpm test
pnpm build
pnpm qa:api
pnpm qa:browser
```

D1 schema 파일이 생긴 뒤에는 로컬부터 적용한다.

```powershell
npx wrangler d1 execute prompt-gallery-db --local --file=./db/schema.sql
```

원격 적용은 로컬 검증 후 진행한다.

```powershell
npx wrangler d1 execute prompt-gallery-db --remote --file=./db/schema.sql
```

## 재확인 명령

Cloudflare 리소스 상태가 의심되면 아래 명령으로 다시 확인한다.

```powershell
cd D:/vibe-coding/prompt-gallery
npx wrangler whoami
npx wrangler d1 list
npx wrangler r2 bucket list
```

```markdown

- prompt-gallery-db

- prompt-gallery-previews
```

## 남은 결정

| 결정 | 권장값 | 이유 |
|---|---|---|
| 인증 | Cloudflare Access | 개인용 앱을 공개 앱처럼 만들 필요 없음 |
| R2 공개 여부 | 비공개 | preview는 Worker proxy로 보호 |
| 이미지 저장 | 압축 preview만 저장 | 원본 보관은 저장량과 운영 부담 증가 |
| 환경 분리 | local + production | 개인용 1차 구현에서는 staging 생략 가능 |
| 백업 | 수동 script 우선 | D1 dump + R2 sync를 나중에 하나로 묶기 쉬움 |
