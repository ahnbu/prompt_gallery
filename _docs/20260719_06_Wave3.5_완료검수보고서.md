## 3. 한계점 및 트레이드오프 검토 (Advisory)

계획에서 '남은 검증 한계'로 명시된 두 가지 항목에 대한 판정입니다.

1. **드래그앤드롭 (DnD) headless 검증 부재**
   - **평가**: Playwright의 native DnD 지원 한계로 인해 E2E 스크립트에서 제외되었으나, `ImagePreviewField.tsx`의 `handleDrop` 이벤트 내부 로직(유효 파일 필터링, 업로드 상태 전이 등) 및 Typecheck가 충분히 견고하게 구현되어 있습니다. E2E 검증 생략은 계획 당시 설계된 합리적 트레이드오프로 판단됩니다.
   - **판정**: **Advisory (권고사항)**. 배포 시 브라우저 내에서 직접 드래그 앤 드롭 파일 첨부를 수동으로 한 번 점검해볼 것을 권장합니다 (Blocker 아님).

2. **orphan cleanup (3-1b) 런타임 검증 부재**
   - **평가**: 신규 아이템(`kind: "add"`)이 staged 이미지 자산을 남기는 버그를 막기 위해 `cleanupCurrentStagedAsset` 함수에서 조기 반환 조건을 `detail`일 때만 적용하도록 변경했습니다. 브라우저 통합 시뮬레이션 없이도 코드상 자산 삭제 API가 호출됨이 논리적으로 명확히 입증됩니다.
   - **판정**: **Advisory (권고사항)**. E2E 테스트 부재는 타당하며 구현 상의 Blocker로 작용하지 않습니다.
