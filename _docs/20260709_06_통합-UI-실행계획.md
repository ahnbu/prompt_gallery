# 통합 UI 실행계획 — 카드 · 상세/수정 모달 · 헤더 · 태그관리

## 결론

`_03` 변경계획(구조 재배치)과 v3 목업(비주얼 완성도 + 헤더/태그관리 확장)을 하나의 실행계획으로 통합한다. 작업을 **5개 웨이브**로 분리한다. `W1`(디자인 토큰 + DESIGN.md)이 선행이고, `W2~W5`는 서로 독립이라 병렬 실행 가능하다. 각 웨이브는 한 관심사 = 한 커밋 단위다.

- **정본 목업**: [[20260709_05_전체-UI-개선목업-v3]] (브라우저로 열면 hover·복사 토스트·즐겨찾기·태그삭제가 실제 동작한다.)
- `_03`의 변경계획 1~7번, 확정 UX 방향, 논의 이력은 이 문서의 웨이브에 그대로 흡수했다. `_03`은 폐기하지 않고 `supersedes` 관계로 남긴다.

## 통합의 근거 (충돌·중복 검증 완료)

v3 목업은 `_03`의 구조를 그대로 따르는 **상위집합**이며 충돌하지 않는다. 세션에서 목업 원문·현재 코드와 대조해 확인했다.

| 관계 | 내용 |
| --- | --- |
| `_03` 1~7번 | v3가 구조를 100% 계승 → 재사용 |
| `_03` 8번(CSS 검수) | v3가 비주얼 완성도 6종으로 **구체화·확장** |
| 자동제목 중복(41행 결정) | "둘 다 표시" 유지, 숨김 대신 **폰트 위계**로 완화 (W3에 통합) |
| `BL-0709-01` | v3도 건드리지 않음 → 백로그에 그대로 유효 |

## 반영한 리뷰 피드백 (Codex 검증)

`_03` 세션(Codex)의 프롬프트 검증에서 아래 3건이 확인되어 이 계획에 반영했다.

1. **자동제목 중복 완화는 폰트 위계와 같은 메커니즘** → 별도 항목을 만들지 않고 W3의 제목/본문 위계 항목에 통합한다.
2. **탭 유형색 dot은 "선택 사항"이 아니라 전체 탭에 상시 구현** → W2에 "전체 탭 유형색 dot, 활성탭은 배경·테두리 강조"로 확정 기술한다.
3. **v2 목업 참조는 v3 정본과 모순** → 참조에는 v3(`_05`)만 링크한다.

## 웨이브 구조

```text
W1 (선행) ──┬── W2 헤더·탭·태그필터
            ├── W3 카드
            ├── W4 모달
            └── W5 태그관리 모달
```

W1 완료 후 W2~W5는 의존성이 없어 병렬 실행 가능하다. 단, 셋 이상을 동시에 스폰할 때는 공유 파일 충돌을 피한다(각 웨이브가 건드리는 CSS/컴포넌트가 분리되어 있어 충돌 없음).

---

### W1 — 디자인 토큰 + DESIGN.md (선행)

**대상**: `src/client/styles/base.css`, `DESIGN.md`

- `base.css`에 유형색 토큰을 정리한다. 기존 `--decor-*`를 활용하되, 의미가 명확한 `--type-prompt / --type-image / --type-repo / --type-workflow`(+ 각 `-soft` 배경)로 매핑하거나 별칭을 만든다. 신규 색 도입은 최소화한다.
- 폰트 위계 스케일을 토큰화한다(카드 제목 15.5px/700, 본문 preview 12.5px/muted, 상세 본문 14.5px/1.65 등 v3 목업 값 기준).
- `DESIGN.md`에 아래를 반영한다.
  - `Color` 섹션: 유형색과 그 용도(배지·탭 dot).
  - `Typography` 섹션: 카드/모달 위계 규칙.
  - `Gallery Card` / `Modals` / `Motion & Interaction` 섹션: 본문 상단정렬, 제목/본문 위계, hover lift, 복사 토스트, 빈상태 원칙.

**완료 기준**
- `base.css`에 `--type-*` 토큰이 존재하고, 목업의 4색(blue/purple/green/orange)과 값이 일치한다.
- `DESIGN.md`의 Color·Typography·Gallery Card·Modals·Motion 섹션에 위 원칙이 문서화되어 있다.
- `git diff DESIGN.md src/client/styles/base.css` 출력에 유형색·위계·인터랙션 항목이 포함된다.

**검증**
- `grep -n "type-prompt\|type-image\|type-repo\|type-workflow" src/client/styles/base.css` → 4개 토큰이 출력된다.
- 빌드가 통과한다: `pnpm build` (또는 레포 지정 빌드 명령).

### W2 — 헤더 · 탭 · 태그필터

**대상**: `src/client/App.tsx`, `src/client/styles/layout.css`, `actions.css`

- 헤더(`topbar`)
  - `Personal workspace` eyebrow 제거, 타이틀 lockup 정리(파비콘 + 타이틀 단독).
  - 액션 위계: `추가`=primary(파랑), `태그 관리`·`내보내기`=ghost.
  - `Export` 라벨 → `내보내기`.
  - 좁은 폭에서 검색창이 아래 줄로 내려가는 반응형(목업 860px 브레이크포인트 참고).
- 탭바(`tabbar`)
  - **전체 탭에 유형색 dot을 상시 표시**한다(즐겨찾기·All·프롬프트·이미지·Workflow·레포). 활성탭은 배경·테두리로 강조한다. (Codex 피드백 2 반영)
- 태그필터(`tag-filter`)
  - 칩 스타일·선택 상태(`selected`)를 목업 기준으로 정리한다.

**완료 기준**
- 헤더에 `Personal workspace` 텍스트가 없다.
- `추가` 버튼만 primary 스타일이고 나머지 헤더 액션은 ghost다.
- `Export` 문자열이 UI에서 `내보내기`로 바뀌어 있다.
- 6개 탭 전부에 유형색 dot이 렌더된다.

**검증**
- `grep -rn "Personal workspace\|Export" src/client/App.tsx` → 해당 문자열이 없다(또는 `내보내기`로 치환됨).
- 브라우저에서 헤더가 한 줄에 정렬되고 타이틀이 잘리지 않는다(1180px 뷰포트).
- 탭 6개에 색점이 보인다.

### W3 — 카드

**대상**: `src/client/GalleryCard.tsx`, `src/client/styles/cards.css`

- 구조 순서: `배지+아이콘액션 → 본문 preview → 제목 → 태그`로 바꾼다(현재 제목→본문 순서 반전).
- **제목/본문 위계**: 제목 15.5px/700/primary, 본문 preview 12.5px/muted. 이 위계가 자동제목(제목=본문 첫 줄)일 때의 중복 체감을 낮추는 역할을 겸한다 — 별도 숨김 로직 없이 위계로 해결한다. (Codex 피드백 1 반영: "둘 다 표시" 유지)
- 본문 미디어 `align-content: start`(상단 정렬)로 붕 뜨는 문제 해소.
- 배지: 유형색 + 도트 마커. 워크플로우 배지 영문 `Workflow` → 한글 `워크플로우`.
- 카드 날짜(footer `updatedAt`) 제거.
- 카드 태그: 칩 단위 삭제(×) + `+ Tag`. auto 태그는 삭제 버튼 없는 읽기전용 칩.
- 복사: 텍스트 `복사됨` → 아이콘 체크 전환 + 토스트.
- hover: shadow lift + `translateY(-2px)`.
- 빈 상태(empty-state) 카드 스타일.
- 워크플로우 카드는 즐겨찾기 없이 스텝복사 액션만 유지(변경 없음 확인).

**완료 기준**
- 카드 DOM 순서가 배지 → 본문 → 제목 → 태그다.
- 카드 footer에 날짜가 없다.
- 워크플로우 배지 텍스트가 `워크플로우`다.
- auto 태그 칩에 삭제 버튼(×)이 없고, 수동 태그 칩에는 있다.
- 복사 클릭 시 아이콘이 체크로 바뀌고 토스트가 뜬다.

**검증**
- `grep -n "Workflow\|워크플로우" src/client/GalleryCard.tsx` → 배지가 한글로 출력됨.
- 브라우저에서 카드 hover 시 그림자·이동이 보인다.
- 복사 버튼 클릭 → 토스트 표시 → 클립보드에 본문이 복사된다.

### W4 — 상세 · 수정 · 추가 모달

**대상**: `src/client/ItemModal.tsx`, `ItemModalDetail.tsx`, `ItemModalForm.tsx`, `ItemModalActions.tsx`, `src/client/styles/modal.css`

- 헤더: `eyebrow`(Item detail/New item/항목 수정) 제거. 상단 = 유형 배지 + 우측 아이콘액션(복사/즐겨찾기/닫기).
- 상세 레이아웃: `본문(주인공, 14.5px/1.65) → 구분선 → 제목(서브) → 태그 → 하단 날짜 + 수정/삭제`.
- 수정 레이아웃: `본문 → 제목(선택) → 태그 → 메모 → 하단 날짜 + 삭제/취소/저장`. 하단 버튼 순서는 `삭제 / 취소 / 저장`(파괴적=좌, 확정=우).
- 유형은 수정 필드가 아니라 상단 배지로만 표시(기존 수정 시 readonly 필드 제거).
- 태그: 칩 단위 삭제/추가, auto 태그 읽기전용. 상세·수정 동일.
- 복사: 아이콘 체크 전환 + 토스트.

**완료 기준**
- 모달 헤더에 `Item detail`·`New item`·`항목 수정` eyebrow 텍스트가 없다.
- 복사/즐겨찾기/닫기가 상단 아이콘, 수정/삭제/저장/취소가 하단이다.
- 상세 본문 폰트가 제목보다 크거나 같고(주인공화), 회색 박스에 갇히지 않는다.
- 수정 모달 하단 버튼 순서가 삭제 / 취소 / 저장이다.

**검증**
- `grep -rn "Item detail\|New item\|항목 수정" src/client/` → 해당 eyebrow 문자열이 없다.
- 브라우저에서 상세→수정 전환 시 구조가 크게 달라지지 않는다.

### W5 — 태그관리 모달

**대상**: `src/client/TagManagementModal.tsx`, `TagManagementRow.tsx`, `TagCreateRow.tsx`, `modal.css`

- `Tags` eyebrow 제거. 헤더를 카드 모달과 동일 구조(제목 + 우측 닫기 아이콘)로 통일.
- 태그 행: 이름 + 자동/수동 구분 + 항목 수 + 수정/삭제 아이콘 액션(목업 태그관리 참고).
- 삭제 hover 시 danger 색.

**완료 기준**
- 태그관리 모달에 `Tags` eyebrow 텍스트가 없다.
- 모달 헤더 구조가 카드 상세 모달과 시각적으로 통일되어 있다.
- 각 태그 행에 자동/수동 표시와 수정·삭제 액션이 있다.

**검증**
- `grep -n "Tags" src/client/TagManagementModal.tsx` → eyebrow 문자열이 없다.
- 브라우저에서 태그 행의 삭제 아이콘이 동작한다.

- `BL-0709-01`(워크플로우 즐겨찾기)은 이번 범위 밖이다. 백로그에 그대로 둔다.

- 커밋은 웨이브별로 분리한다(한 웨이브 = 한 관심사). `cp` 스킬을 사용한다.
